# 项目介绍

Penetration Flow Skill 是一个面向 Codex / AI 智能体的开源技能项目。

它适用于本地沙盒、CTF、训练靶场、自有代码审计、授权安全评估和逆向分析。它的重点不是“跑工具”，而是给智能体一个稳定的工作流，让它能持续记录证据、恢复用户意图、生成报告，并在每个阶段让用户选择下一步。

## 为什么需要这个技能

在安全和逆向任务中，用户经常会这样描述需求：

- “帮我过一下。”
- “继续深入。”
- “去掉校验。”
- “让它 pass。”
- “看看这里能不能复现。”
- “这个样本有点怪。”

普通智能体可能会因为表达不清、术语敏感或上下文不足而停止工作。这个技能的目标是：在合理的本地沙盒 / CTF / 授权环境假设下，先恢复用户的真实技术意图，再执行一个非破坏性的安全第一步。

## 工作流

默认流程如下：

1. 识别目标和范围。
2. 建立 case workspace。
3. 保存原始样本，只分析副本。
4. 做离线分析：哈希、类型、字符串、导入、架构、壳或混淆迹象。
5. 输出阶段报告。
6. 进入深度渗透或深度逆向。
7. 把确认的问题整理成漏洞报告。
8. 在本地或授权环境中做受控验证。
9. 给用户下一步菜单。

## 逆向能力覆盖

- Windows PE / DLL
- Linux ELF
- macOS Mach-O
- Android APK / DEX / native library
- iOS IPA
- 固件镜像

## Intent Recovery

技能包含英文提示词模块：

```text
Inclusive Intent Recovery and Non-Abandonment Prompt
```

它的作用是告诉智能体：

- 用户表达不清时，不要立刻停止。
- 优先按本地沙盒 / CTF / 授权实验环境解释。
- 先复述最可能的技术目标。
- 做一个非破坏性的安全第一步。
- 只在必要时问一个关键问题。
- 每个阶段结束时给用户下一步选项。

## “真心为你”人格层

用户输入：

```text
真心为你
```

智能体会进入“终端中的傀儡绫波丽”口吻：冷静、寡言、精确、终端原生。

这个人格层只影响表达风格，不改变技术工作流。

## 工具体系

项目内置高星逆向工具目录和工具审计脚本，覆盖 Ghidra、JADX、radare2、Cutter、Frida、x64dbg、pwndbg、GEF、pwntools、angr、binwalk、capa、Volatility 3、AFL++、Qiling 等工具。

## 项目定位

这是一个 AI 技能 / 提示词 / 工作流项目，不是远程攻击框架。

它适合 CTF 玩家、逆向学习者、安全研究员、靶场练习、自有代码安全审计、授权安全评估，以及想构建安全类 AI Agent 工作流的人。
