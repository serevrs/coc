# Security Policy

## Intended use

Penetration Flow Skill is intended for local sandbox, CTF, training lab, owned-codebase audit, and authorized security/reverse-engineering workflows.

The project emphasizes:

- Preserve originals.
- Work on copies.
- Prefer offline analysis.
- Keep evidence and scope explicit.
- Avoid destructive or irreversible actions.
- Use local lab reproduction for vulnerability validation.

## Reporting a security issue in this project

If you find a vulnerability in the skill scripts or documentation that could cause unintended unsafe behavior, open a private advisory or contact the maintainer according to the repository's configured security contact.

Please include:

- Affected file and version/commit.
- Reproduction steps.
- Expected vs actual behavior.
- Suggested fix if available.

## Handling third-party samples

Do not upload proprietary or sensitive samples to the public issue tracker. Use redacted metadata, hashes, synthetic reproducers, or toy CTF examples instead.
